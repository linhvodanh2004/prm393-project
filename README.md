# prm393_project — Flutter + Firebase

## TL;DR

- **What**: Flutter mobile app for room discovery and hourly booking, with role-based flows (User / Host / Admin).
- **Backend**: Firebase (Auth + Firestore + Cloud Messaging). Supporting services: Cloudinary (images), Vietnam Provinces API (location).
- **Docs**: Detailed setup lives in `SETUP.md`.

## DTO Pattern

All write operations go through DTOs instead of directly constructing model objects. Each DTO:

- Defines only the fields required by the caller
- Provides a `validate()` method returning an error string or `null`
- Provides `toModel()` to convert to the corresponding Firestore model

### DTOs and their service targets

| DTO | Service method |
|---|---|
| `RegisterDTO` | `AuthService.registerWithDetails` |
| `CreateBookingDTO` | `BookingService.createBooking` |
| `UpdateBookingStatusDTO` | `BookingService.updateBookingStatus` |
| `CreateNotificationDTO` | `NotificationService.createNotificationFromDto` |
| `CreateRoomDTO` | `RoomService.createRoom` |
| `UpdateRoomDTO` | `RoomService.updateRoom` |
| `SetDailyPriceDTO` | `RoomService.setDailyPrice` |
| `CreateVoucherDTO` | `VoucherService.createVoucherWithRandomCode` |
| `SendMessageDTO` | `ChatService.sendMessage` |
| `SubmitHostRequestDTO` | `HostRequestService.submitRequest` |
| `SavePropertyDTO` | `PropertyService.saveProperty` |

## Data Model (Firestore)

### Collections overview

| Collection | Description |
|---|---|
| `users/{uid}` | User profile, role, FCM token, favorites list |
| `rooms/{roomId}` | Host room listings |
| `rooms/{roomId}/daily_prices/{yyyy-MM-dd}` | Per-date price overrides and block flags |
| `bookings/{bookingId}` | Booking records with status machine |
| `notifications/{notificationId}` | In-app notification inbox |
| `chat_rooms/{chatRoomId}` | 1:1 chat room metadata |
| `messages/{messageId}` | Chat messages |
| `host_requests/{requestId}` | "Become a host" applications |
| `properties/{hostId}` | Host property/hotel profile (doc ID = hostId) |
| `vouchers/{voucherId}` | Discount voucher definitions |
| `voucher_redemptions/{redemptionId}` | Per-user voucher usage tracking |

```json
{
```

```json
  "images": ["cloudinaryUrl"],
  "basePrice": 100000.0,
```

### `rooms/{roomId}/daily_prices/{yyyy-MM-dd}`

```json
{
  "roomId": "string",
  "date": "Timestamp",
  "price": 120000.0,
  "isBlocked": false
}
```

> When `isBlocked: true`, bookings spanning that day are rejected at the UI level.
> When `isBlocked: false` and a price override exists, it replaces `room.basePrice` for every hour on that date.

### `bookings/{bookingId}`

```json
{
  "roomId": "string",
  "roomTitle": "string",
  "userId": "string",
  "userName": "string",
  "hostId": "string",
  "checkIn": "Timestamp",
  "checkOut": "Timestamp",
  "guestCount": 1,
  "totalPrice": 240000.0,
  "status": "pending | confirmed | rejected | paid | completed | cancelled",
  "note": "string?",
  "createdAt": "Timestamp",
  "updatedAt": "Timestamp",

  "voucherId": "string?",
  "voucherCode": "string?",
  "voucherScope": "HOST | GLOBAL?",
  "voucherHostId": "string?",
  "voucherDiscountAmount": 20000.0
}
```

#### Booking status machine

```
pending ──► confirmed ──► paid ──► completed
        ↘   ↘              ↘
         cancelled      cancelled   (confirmed can also go directly to completed)
        rejected (terminal)
```

### `vouchers/{voucherId}`

```json
{
  "code": "AUTOGENERATED",
  "scope": "HOST | GLOBAL",
  "hostId": "string?",
  "type": "PERCENT | FIXED",
  "value": 10,
  "maxDiscount": 50000,
  "minSubtotal": 100000,
  "startAt": "Timestamp?",
  "endAt": "Timestamp?",
  "isActive": true,
  "usageLimitTotal": 9999,
  "usageLimitPerUser": 1,
  "createdBy": "string",
  "createdAt": "Timestamp",
  "updatedAt": "Timestamp"
}
```

**Voucher code**: always auto-generated by `VoucherService.generateUniqueVoucherCode()` (10-char alphanumeric, excludes `I O 0 1`). Hosts/admins cannot manually enter codes.

**Voucher scope rules**:
- `HOST`: valid only when `booking.hostId == voucher.hostId`
- `GLOBAL`: valid for any booking on the platform

**Minimum subtotal validation**: `subtotal >= max(voucher.minSubtotal, discountCap)` where `discountCap` is `voucher.value` for FIXED type or `voucher.maxDiscount` for PERCENT type.

### `voucher_redemptions/{redemptionId}`

```json
{
  "voucherId": "string",
```

### `chat_rooms/{chatRoomId}`

```json
{
  "participants": ["uidA", "uidB"],
  "participantNames": { "uidA": "Alice", "uidB": "Bob" },
  "participantAvatars": { "uidA": "https://...", "uidB": "https://..." },
  "lastMessage": "string",
  "updatedAt": "Timestamp",
  "unreadCounts": { "uidA": 0, "uidB": 2 }
}
```

**Constraint**: Only 1:1 private chats. `participants.length == 2`.
**Room ID strategy**: `<min(uidA, uidB)>_<max(uidA, uidB)>` — deterministic to prevent duplicate rooms.

## Capabilities (by role)

### Navigation (all roles)

All roles display a **profile avatar** in the AppBar (top-right), replacing the old footer "Hồ sơ" tab. The avatar shows:
1. The user's photo from Firestore `photoURL` (falls back to `FirebaseAuth.currentUser.photoURL`)
2. Two-letter initials if no photo is set (e.g. "Nguyễn Văn A" → "NA")
3. A generic person icon if no name is available

Tapping the avatar pushes `UserDetailsScreen` as a modal route with a back button.

### User

**Authentication**
- Email/password registration + Google Sign-In
- Forgot password (email recovery)
- Account lockout: login is blocked if `user.isActive == false`

**Explore & Discovery**
- Browse available rooms (status = `available` only)
- Filter by amenities and price
- Search by room title and address
- View host's property address on room cards

**Room Details & Booking**
- Image carousel, description, amenities
- View **host public profile** (property info, vouchers, other rooms, quick message)
- Booking flow: select check-in and check-out datetime separately
- Price calculation: hour-by-hour, respecting `daily_prices` overrides per date
- Blocked date detection: if any hour in the selected range falls on a blocked day, booking is rejected with a clear error
- Apply voucher code during checkout (validated against scope, expiry, per-user limit, min subtotal)
- Booking status tracking: Pending → Confirmed → Completed / Cancelled

**Vouchers (dedicated tab)**
- Browse active global vouchers ("Toàn sàn") and host-specific vouchers ("Đối tác")
- Copy voucher code to clipboard
- Shows host name for host-specific vouchers

**Chat**
- Search users by name, email, or phone
- Start or resume 1:1 private chat
- Unread message badge on chat icon

**Notifications**
- In-app notification inbox
- Unread notification badge on bell icon

### Host

- **Room management**: create, edit, delete rooms via `CreateRoomDTO` / `UpdateRoomDTO`
- **Booking operations**: confirm, reject, complete guest bookings; direct chat with guests
- **Calendar/pricing**: set per-day price overrides and block dates via `SetDailyPriceDTO`; pricing is hourly
- **Revenue dashboard**: monthly/yearly stats with bar chart (confirmed + paid + completed bookings)
- **Voucher management**: create host-scoped vouchers (code is auto-generated)
- **Property profile**: update hotel name, description, address, cover image via `SavePropertyDTO`

### Admin

`ManageUsersScreen` has three sub-tabs with live stat bars (`AdminStatCard`):

| Sub-tab | Content |
|---|---|
| Người dùng | All users; tap a card to see a detail popup (name, role, email, phone, status, join date); lock/unlock/promote actions via overflow menu |
| Đối tác | Only HOST-role users; live stat bar; tap a card to fetch and display the host's `properties` doc in a detail popup |
| Yêu cầu | Pending "become a host" applications; approve (auto-creates `properties/{userId}`) or reject with a reason note |

Other admin screens:

- **Room management** (`AdminRoomsScreen`): stat bar (total / available / pending / inactive); filter by status chip; title+address search; tap a room card for a detail popup; approve or reject rooms
- **Booking oversight** (`AdminBookingsScreen`): stat bar (total / pending / confirmed / completed / cancelled) in the "Booking" tab; embedded `AdminRevenueScreen` in the "Doanh thu" tab; force-complete or force-cancel any booking
- **Revenue** (`AdminRevenueScreen`): platform-wide revenue with status and date-range filters; can run standalone or embedded in the Booking tab

## Notifications

### Surfaces

- **In-app inbox**: Firestore `notifications` collection filtered by `recipientId`
- **Device push**: FCM push to `users/{uid}.fcmToken`, shown via `flutter_local_notifications`

### Delivery rule

Write in-app notification first, then attempt FCM push (push failure does not lose the in-app notification).

### Trigger matrix

| Event | Who gets notified |
|---|---|
| Booking created | Host (in-app + device) |
| Booking status changed by host/admin | User (in-app + device) |
| Booking cancelled by user | Host (in-app + device) |
| New chat message | Recipient (in-app + device) |
| Host request submitted | Admin (in-app + device) |
| Host request approved/rejected | User (in-app + device) |

### Unread badges

- **Chat icon** (`ChatBadgeIcon`): streams `ChatService.getTotalUnreadCount()` — sum of `unreadCounts[currentUid]` across all chat rooms
- **Bell icon** (`NotificationBadgeIcon`): streams `NotificationService.getUnreadCount()` — count of unread notifications

## Hourly Pricing and Daily Overrides

Rooms are priced **per hour**. The total price for a booking is calculated by iterating hour-by-hour:

```
for each hour h in [checkIn, checkOut):
    dayKey = date(h)
    if daily_prices[dayKey] exists:
        if daily_prices[dayKey].isBlocked → reject entire booking
        else → price += daily_prices[dayKey].price
    else:
        price += room.basePrice
```

Daily price overrides are stored at: `rooms/{roomId}/daily_prices/{yyyy-MM-dd}`

## Configuration

### Firebase

- **Android**: `android/app/google-services.json`
- **iOS**: `ios/Runner/GoogleService-Info.plist` + APNs certificate for device push
- **Generated options**: `lib/firebase_options.dart` (run `flutterfire configure` to regenerate)
- **Auth providers**: Email/Password + Google Sign-In (see `SETUP.md` for SHA-1 steps)

### Environment variables (`.env`)

The app uses `flutter_dotenv`. Create a file named `.env` in the root directory and add the following keys:

```env
CLOUDINARY_CLOUD_NAME=your_cloudinary_cloud_name
CLOUDINARY_API_KEY=your_cloudinary_api_key
CLOUDINARY_API_SECRET=your_cloudinary_api_secret
```

> Do not commit `.env` to version control (it is ignored in `.gitignore`). Add a `.env.example` with placeholder values to help other developers setup their environments.
